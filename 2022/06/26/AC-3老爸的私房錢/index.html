ALPHA Camp 網頁記帳app專案-老爸的私房錢

Posted on 2022-06-26 In Project

ALPHA Camp 學期三後端班 Week 2 指標作業

為何會選擇這個專案？

從頭開始建立記錄支出的專案，除了可以熟悉一路所學習的後端基本功，包括資料CURD操作、樣版引擎、使用者認證，也加入了資料關聯性（支出類別）以及呈現篩選資料的操作練習。

使用了什麼技術？

  • Language: JavaScript
  • Runtime environment: Node.js
  • Framework: Express
  • Template engine: Express-handlebars
  • Database: MongoDB Altas & mongoose
  • User authentication: Passport.js
  • 使用非同步async/await處理Promise，在某些route有複雜的資料處理時，寫起來更加直覺順暢。

哪部分你相對能掌握？哪裡花了最多時間？

能掌握：

  • Mongoose連線、樣版引擎設定與view建立（搭配Bootstrap）
  • Mongoose schema model建立、資料類型、validator
  • 基本CRUD routes建立，response傳入需要的物件供樣版引擎使用
  • Mongoose model中CRUD常用methods (.find, .findByIdAndUpdate 等等)
  • Middle ware概念
  • 將使用者密碼加密後再存入資料庫

能理解並實作，但還不是很熟悉：

  • Cookie & Session以及使用Passport做使用者認證功能（註冊、登入登出）
  • JavaScript: commonJS require/export ，在route, model, function上不同的寫法

花很多時間：

  • Seeder資料依照3個table的資料關聯性去建立，看似簡單但也反覆修改
  • 在使用類別下拉式選單中，為了不把類別資料寫死在前端樣版上，在各項與類別有關的操作上都花較多時間寫
  • 跑Seeder的script，原本Promise語法寫得舉步維艱，改成async/await後才比較順暢
  • 資料關聯性到資料結構的設計，覺得應該有更系統性的做法，而不是先做我直覺知道作法的，再去修改。期待接下來課程中「關連式資料庫」與SQL的學習內容，能讓我對於資料的操作更佳的順暢

過程中碰到什麼困難？又如何克服？（例如：查找網路文件）

  1. 由於專案設計有很多使用類別(Category)資料的地方，但想只用db給的_id來做資料關聯性，不想在其他table中增加多的欄位，造成使用Category時（例如要render該類別的icon、首頁要可以篩選類別），都要一直反覆查找_id來找到關聯資料，不曉得這樣的設計會不會有什麼潛在的問題？
  2. 類別下拉式選單是將類別物件整包丟到前端樣版，再用{{#each}}渲染出每個選項的，所以在做首頁的篩選功能、以及編輯頁面處理類別資料時，想要保持傳入的該筆資料類別selected。查找資訊發現handlebars-helper套件可以擴充handlebars的render條件，安裝後試了{{#is}}功能，若條件為單純string或傳入值都可以，但放在{{#each}}中就沒能成功，應該不是不能nesting，可能需要看helper的原始語法才能略知一二。此部分先暫停。
  3. Mongodb的日期格式是包含時區的完整資訊，但因為只想在頁面上顯示簡單的年月日，查找網路資訊有相關套件dayjs可以使用，但也發現簡單的date.toString()就可以得到YYYY-MM-DD格式，所以雖然又存了一個簡易日期的資料欄位，但就先從簡處理。

過程中你有對哪個技術有特別深刻的學習？

  1. 我對使用者認證、以及如何加密使用者的敏感資訊這一塊很有興趣。在學習以及實作認證功能中，特別花時間去理解每一步驟的資料傳遞、所經過設定的middle ware、OAuth中與第三方的溝通，以及Passport.js套件所提供的方法應用。
  2. 在學習Promise與async/await的過程中，對於物件的概念越來越清楚，也可以從各項工具中看到物件的身影（例如mongoose schema的建立語法），期待自己後續能更深入熟悉JavaScript語法。
